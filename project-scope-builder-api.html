A backend API designed to generate clear, defensible bug bounty scope documents by combining automation, heuristics and AI.

Reduces manual effort while keeping humans firmly in control of decisions.

Backend API AI Integration System Architecture Security Configuration-Driven

What It Does

Accepts a company domain and builds a structured security testing scope across:

  • Websites and subdomains
  • APIs and documentation endpoints
  • Mobile applications (iOS and Android)

Produces consistent, explainable outputs suitable for security programs and auditors.

Key Design Strengths

Subdomain Intelligence, Not Just Discovery

Uses Subfinder for discovery, then filters, normalises, groups and enriches results so they are meaningful for downstream decisions rather than raw lists.

AI Guided by Structure and Constraints

AI is used after data has been cleaned, deduplicated and framed. This dramatically improves recommendation quality and avoids "AI guesswork."

Mobile App Discovery with Controlled Ambiguity

App store searches are inherently noisy. To manage this, the API supports multiple explicit search modes:

  • APP NAME - Search by application name only
  • DEVELOPER - Search by developer/publisher
  • EITHER - Broader discovery (match either criteria)
  • BOTH - Precision matching (match both criteria)

This makes the trade-offs visible instead of hiding them behind a single opaque search.

Shared Architecture Across Platforms

Initial AI-generated implementations for iOS and Android diverged. The solution was refactored to use a shared delegate pattern:

  • Reduced duplication and cut the codebase from ~300 lines to ~150
  • Improved maintainability
  • Prevented platform-specific design drift

Heuristics Before AI for API Discovery

Likely API and documentation endpoints are identified, grouped and scored first. AI is then used to refine naming and recommendations, not to blindly search the web.

Config-Driven and Extensible

Behaviour is controlled via configuration rather than hard-coded rules, making the system easy to evolve, test and extend without breaking existing logic.

Why It Matters

Architectural Control with AI

This project demonstrates strong architectural control when working with AI: knowing when to lean on automation, when to constrain it and when to override it with deliberate design decisions.

The result is a system that is reliable, explainable and practical in real-world security programs.

Rather than treating AI as a magic solution, this API shows how to combine deterministic logic, structured data and constrained AI reasoning to produce consistent, defensible outputs that security teams can trust.
