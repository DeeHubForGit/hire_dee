Designed and built an AI-assisted web application that generates high-quality bug bounty scope documents from a company domain.

The tool automates what is normally a manual, error-prone process by combining deterministic data retrieval with carefully constrained AI reasoning.

AI Integration API Design Web Development System Architecture Configuration-Driven Design

What It Does

  • Retrieves subdomains, mobile applications and API endpoints for a target organisation
  • Uses heuristics first, then AI, to produce accurate and practical scope recommendations
  • Generates a structured, ready-to-use bug bounty scope document via a guided UI

Key Technical Strengths Demonstrated

API Design & Ownership

Built and used my own API after years of integrating with third-party APIs, gaining full end-to-end visibility into API design, validation and error handling.

AI with Architectural Control

AI is used where it adds value, not as a black box. Inputs are cleaned, constrained and structured to produce reliable outputs.

Complex Mobile App Discovery Logic

App store searches are inconsistent across regions and platforms.

  • Designed multiple search modes (APP NAME, DEVELOPER, EITHER, BOTH) to balance recall and accuracy across iOS and Android
  • Handles platform-specific quirks and edge cases systematically

Shared Logic & Maintainability

When AI produced near-duplicate logic for iOS and Android, I stepped in to introduce a shared delegate.

  • Reduced code size by ~50%
  • Prevented long-term design drift
  • Improved maintainability and consistency

Configuration-Driven Design

Behaviour is driven by configuration files rather than hard-coded rules, making the system easier to extend, tune and maintain.

Why It Matters

Real Business Impact

Reduces manual effort and cognitive load for security teams by automating repetitive reconnaissance work.

Produces more accurate scopes by combining business context, technical heuristics and AI reasoning in a structured way.

Demonstrates strong technical business analysis, system design and AI-assisted development discipline. Shows how to use AI effectively while maintaining control over architecture, quality and maintainability.

Technical Approach

This project showcases a pragmatic approach to AI integration:

  • Deterministic first: Use reliable, rule-based logic wherever possible
  • AI where valuable: Apply AI for pattern recognition and context-aware decisions
  • Constrained inputs: Clean and structure data before feeding it to AI
  • Human oversight: Review AI outputs and step in when architecture matters
  • Maintainable design: Prioritise code quality and long-term maintainability over quick wins
